Buenas prácticas:

1 - Describir el problema
    cuál es el input.
    cuál debe ser el resultado.
    qué lo impide?

2 - Reproducir el error
    Algunos errores ocurren en casos específicos.
    Crear la instancia en que el error ocurre.

3 - Leer como la computadora.
    Seguir la ruta lógica del proceso

4 - Arreglar errores mostrados por IDE
    Usar try, except u otros métodos para informar a usuario de error

5 - Usar print para visualizar variables no impresas

6 - Usar debugger (como el punto 3)
    Algunos IDE muestran las variables y sus valores en tiempo real
